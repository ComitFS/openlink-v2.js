CAS Teams API

This API enables a developer to use MS Teams as a CAS telephone service component (TSC).

This API does not automate or control the MS Teams client directly as a soft-phone. Instead, it uses the MsTeams class which can be embedded into an application like the demo CAS CRM application to create an ACS endpoint for each MS Teams user using their MS Teams identities. It effectively turns the host application into another instance of MS Teams (like the mobile app). The host application can be used to answer or make phone calls on behalf of the end user's Teams provisioned PSTN number and have the same user interface optimized for a financial adviser just like a Cisco phone user.

It processes CAS Openlink commands and routes them to the appropriate ACS endpoint which uses the Microsoft Azure comminications-calling API to perform the expected telephony request. It also recieves the Azure comminications-calling API events from the client library and translates them back into CAS Openlink events which are published to the CAS pusub engine and can them be consumed by subscribing users and applications.

In order to enable this, CAS needs to be configured with approriate permissions. See Microsoft documentation for more details.

There is a demo CRM web application to test this. Ensure the web app url is also registered as a callback url in Azure as part of the configuration. The web app will respond to the commands from the CAS Admin UI and feedback events.

This API can also be used to create ordinary ACS endpoints for non MS Teams users that are anonymous or have an identity from an external directory supported by CAS.

These endpoints can be used in customer facing applications like the CAS Institional Client demo to establish direct voice calls from customers to Ms Teams users and bypass the use of PSTN. These media rich conversations can include screen or desktop sharing and video communication with multiple participants.

There is a demo customer web application to test this. Ensure the web app url is also registered as a callback url in Azure as part of the configuration. The web app will respond to the commands from the CAS Admin UI and feedback events.
